# Ruster

Rust+Router

## コンセプト
ネットワークの学習

### 設定ファイル
- ある設定間の依存関係が分かりにくくなる
- 非明示的に設定される設定内容

以上のような設定の不透明化を助長する行為は決して行わない

実現のため設定に原則としてCLI等の部分単位の変更を許容せず，設定ファイルを主に活用する.

また，非明示的な設定が行われることを防ぐために，lockファイルの考えを活用する．
非明示的な設定内容を全て記述することは非常に煩雑である．
そのため，明示的な設定を行うファイルと，その設定ファイルから生成されるlockファイルという2つの設定ファイルを使用する

また，lockファイルの生成において積極的にWarning, Errorを提示する．
このことにより，設定を適用する前段階のlockファイルの生成段階において設定間の依存関係の不足や設定の抜けを認識できるためである


### 性能目標
一般家庭向けに普及している10Gbpsを目標とする

### L2以上はカーネルに頼ることを行わない

L1は，ハードウェア固有の設定が多く，今回の主眼ではない，そのためLinuxの機能を最大限活用する

逆にL2以降はカーネルは使用せず，本ソフトウェアでL2スイッチングも含めて処理する

L2フレームの取得には，DPDK, AF_PACKET, AF_XDP等を複数選択可能にする

### フルスクラッチ

原則として，ネットワーク的部分(パケット分析，ルーティング)において外部ライブラリを活用せず独自実装を行う
これはネットワークの学習を実現するためである

### Containerlabの活用

テスト，検証段階においてContainerlabを活用する．
Containerlabはトポロジをテキストで定義でき，CIとの相性が良い

cargo test等で活用を行う
